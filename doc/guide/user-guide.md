# 用户指南

本指南面向平台管理员与运营人员。

## 角色说明

- 管理员：维护项目、版本、公告、反馈与日志
- API 调用方：使用 API Key 调用对外接口

## 常见操作流程

## 1. 新建项目

1. 登录后台
2. 进入项目管理
3. 填写项目 key、名称、仓库地址等信息

## 2. 发布版本

1. 进入指定项目
2. 填写版本号、发布说明、下载链接
3. 根据需求设置 `latest` 或 `preview`

## 3. 发布公告

1. 进入公告管理
2. 绑定项目并编辑正文
3. 发布后前台与 API 可见

## 4. 处理反馈

1. 进入反馈管理
2. 按状态筛选待处理项
3. 更新状态并记录处理结论

## API Key 使用建议

- 为不同系统生成独立 Key，便于追踪与撤销
- 设置合理过期时间，避免长期暴露风险
- 仅授予必要权限与项目范围

## 安全与审计建议

1. 定期轮换敏感凭据
2. 监控异常请求频率
3. 对关键管理操作保留审计记录
